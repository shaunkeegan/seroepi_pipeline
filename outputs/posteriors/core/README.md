# outputs/posteriors/core/

Full posterior draw files for models fit to the full dataset (`core` mode). Generated by `02_model_process.R`. All files are excluded from version control.

## File naming convention

Files use opaque IDs: `YYYYMMDD-NNNNNN.rds` with a paired `YYYYMMDD-NNNNNN.json` sidecar.

Use `find_file_by_metadata()` in `R/utils.R` to locate files by antigen, subset, and count type.

## JSON sidecar metadata fields

| Field | Description |
|-------|-------------|
| `id` | Unique file ID (`YYYYMMDD-NNNNNN`) |
| `created` | ISO 8601 timestamp of file creation |
| `mode` | `"core"` |
| `antigen` | `"K"`, `"O"`, or `"KO"` |
| `subset` | `"Full"`, `"Carba"`, `"ESBL"`, or `"Fatal"` |
| `count_type` | `"adj"` or `"raw"` |
| `source_file` | Input site count filename |
| `n_draws` | Number of posterior draws retained |

## RDS contents

Each `.rds` file is a data frame with one row per posterior draw × locus × subgroup, containing:

| Column | Description |
|--------|-------------|
| `locus` | Serotype locus identifier |
| `subgroup` | WHO region |
| `.draw` | Draw index |
| `prevalence` | Posterior draw value on the probability scale |
