# outputs/estimates/LOO/

Summary posterior statistics for leave-one-out models — one model per held-out study. Generated by `02_model_process.R`. All files are excluded from version control.

## File naming convention

Files use opaque IDs: `YYYYMMDD-NNNNNN.csv` with a paired `YYYYMMDD-NNNNNN.json` sidecar.

Use `find_file_by_metadata()` in `R/utils.R` to locate files by antigen, subset, count type, and held-out study.

## JSON sidecar metadata fields

| Field | Description |
|-------|-------------|
| `id` | Unique file ID (`YYYYMMDD-NNNNNN`) |
| `created` | ISO 8601 timestamp of file creation |
| `mode` | `"LOO"` |
| `antigen` | `"K"`, `"O"`, or `"KO"` |
| `subset` | `"Full"`, `"Carba"`, `"ESBL"`, or `"Fatal"` |
| `count_type` | `"adj"` or `"raw"` |
| `loo_study` | Study ID that was held out (e.g. `"Kilifi"`, `"AIIMS"`) |
| `source_file` | Input site count filename |

## CSV columns

Same as `outputs/estimates/core/` — see that README for column descriptions.
